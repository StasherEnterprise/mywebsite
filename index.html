    • 1. phpMyAdmin: Nieuwe MySQL-gebruiker Aanmaken

    Stappenplan:

    1. Log in op phpMyAdmin:
      • Ga naar http://localhost/phpmyadmin.
      • Log in met je MySQL-beheerdersaccount (bijvoorbeeld de root-gebruiker).
    2. Maak de gebruiker aan:
      • Klik in het menu bovenin op Gebruikersaccounts (Users Accounts).
      • Klik op Voeg gebruiker toe (Add user account).
    3. Voer gebruikersgegevens in:
      • Gebruikersnaam: Kandidaat
      • Host: Verander "Any host" naar "Local". In het veld ernaast staat nu als het goed is: localhost.
      • Wachtwoord: 3xamen2023!
      • Authentication plugin: Native MySQL authentication

        • !Generate password veld niks mee doen!



    4. Database rechten toewijzen:
      • Laat bij Database for user account allebei de opties leeg
    5. Global privileges:
      • Laat bij ""Global privileges" ook alles leeg. Niks mee doen
    6. SSL:
      • Doen we lekker niks mee, gewoon lekker op "Require none" laten staan
    7. Bevestig:
      • Klik onderaan op Go om de nieuwe gebruiker aan te maken.
    8. Rechten:

        1. Als je ziet dat de nieuwe gebruiker juist is aangemaakt, refresh dan de pagina. Ga dan naar Home (Huis icoontje linksboven onder het phpmyadmin logo).

        2. Ga dan weer naar User accounts, nu zie je de nieuwe user erbij staan. Klik hier bij de user Kandidaat op "Edit privileges"

        3. Nu zie je 4 kopjes, "Global" "Database" "Change password" en "Login info". Laat bij "Global" alles leeg en ga naar het kopje "Database"

        4. Hier kun je "Add privileges on the following database:". Selecteer hier "Purple_hrm" en klik op Go.

        5. Nu zit je in "Database-specific privileges". Zet hier het vakje "Check all" aan en klik weer op Go.

        6. Als je ziet dat je succesvol de privileges hebt geupdate, refresh dan weer de pagina en log uit (Deur icoontje naast home icoontje)

        7. Log nu in met de nieuwe gebruikersnaam Kandidaat en wachtwoord 3xamen2023!.

        8. Als je ingelogd bent zie je links bij de databases als het goed is alleen de purple_hrm database. Klap deze uit om te kijken of je de tabellen kunt zien

        9. Als je in Home bent zie je rechts op het scherm onder "Database server" dat je met Kandidaat bent ingelogd.

        10. Maak hier een screenshot waarop dus links de uitgeklapte database en rechts de ingelogde user Kandidaat te zien is. 2 vliegen in 1 klap.

    MySQL Query's

    Zorg dat je in de juiste database zit om de query's te runnen.

    Mogelijk moet je dus weer uitloggen en met de root inloggen om bij een andere database als purple_hrm te komen. Maar als je alleen de purple_hrm nodig hebt kun je gewoon die DB selecteren, op SQL klikken en hier de query's maken. (Zie afbeelding)

    Query 1: Totaal aantal medewerkers in dienst

    SELECT COUNT(*) AS totaal_in_dienst FROM employees

    (Deze tabel kan ook anders heten, bv. "Medewerkers". Klap links op het scherm de database zover uit tot je de naam ziet van de nodige tabel)

    WHERE in_service = 1;

    (De "in_service" is in dit geval de tabel die aangeeft of een medewerker in dienst is, dit heet op het examen waarschijnlijk iets als "Dienst_status".

    De "1" is in dit voorbeeld dat een medewerker in dienst is, maar op het examen heet dit iets als "Actief")

    Query 2: Overzicht van medewerkers in dienst

    SELECT firstname, lastname, adres, city, zipcode, email, CONCAT(firstname, lastname) AS username, 'W@chtwoord123!' AS password, d.name AS department FROM medewerkers m (Op het examen kan deze torrie tabel ook anders heten, bv. "employees". Gebruik dan alsnog de letter m erachter om fouten te voorkomen) JOIN departments d ON m.department_id = d.id WHERE m.in_service = 1; (De "in_service" is de tabel die aangeeft of een medewerker in dienst is, dit kan op het examen ook anders heten bv. "Dienst_status". De "1" is in dit geval dat een medewerker in dienst is, maar op het examen kan dit ook bv. "Actief" zijn.)

    Query 3: Medewerkers uit dienst in 2022

    SELECT firstname, lastname, CONCAT(firstname, lastname) AS username, outservice_date (Dit is de tabel wanneer een medewerker uit dienst is gegaan, op het examen kan dit ook anders heten zoals "einde_dienst") FROM medewerkers (Op het examen kan deze tabel ook anders heten, bv. "employees") WHERE outservice_date BETWEEN '2022-01-01' AND '2022-12-31' ORDER BY outservice_date ASC;

    SQL-Database aanpassen

    1. Maak de tabel adstatus

    CREATE TABLE adstatus ( status_id INT(10) NOT NULL PRIMARY KEY, status_description VARCHAR(255) NOT NULL );

    2. Voeg statussen toe aan de tabel adstatus

    INSERT INTO adstatus (status_id, status_description) VALUES (1, 'In de Active Directory en user is actief'), (2, 'In de Active Directory en user niet actief'), (3, 'Niet in de Active Directory'), (0, 'Onbekend');

    3. Breid de tabel employees uit met het veld adstatus_id

    ALTER TABLE employees ADD adstatus_id INT(10) AFTER last_column;

    4. Maak een relatie tussen employees en adstatus

    ALTER TABLE employees ADD CONSTRAINT fk_employees_adstatus FOREIGN KEY (adstatus_id) REFERENCES adstatus(status_id) ON DELETE SET NULL ON UPDATE CASCADE;

    Query 4: Update de adstatus_id van employee 145 naar 3

    UPDATE employees SET adstatus_id = 3 WHERE employee_id = 145;

    Query 5: Controleer met een DELETE query dat status_id = 3 niet uit de tabel adstatus verwijderd kan worden

    DELETE FROM adstatus WHERE status_id = 3;
